# AI事業開発パートナー「Loopin」クローン開発要件定義書

## 1. プロジェクト概要
**目的:**
ユーザー（起業家・新規事業担当者）がAIと対話（壁打ち）を行うことで、事業アイデアを具体化し、最終的に事業計画書（PDF）と分析レポートを出力するWebアプリケーションを開発する。

**開発コンセプト:**
「AI駆動開発」を前提とし、コード、デザイン、ロゴ画像などのアセット生成のすべてをAI主導で行う。

## 2. デザイン・UI/UXガイドライン
*   **デザインテーマ:** 清潔感のあるモダンなデザイン。
*   **アニメーション:**
    *   要素の出現時やホバー時に「ビヨーン」と弾むような動的な演出（Spring animation等）を多用し、リッチな操作感を提供する [1, 2]。
*   **アセット:** ロゴやアイコン、サムネイル画像はすべてDALL-E 3等のAI画像生成を使用する [2, 3]。
*   **レスポンシブ:** PC/SP両対応。

## 3. ユーザー体験フロー (User Journey)

### Phase 1: オンボーディング & プロジェクト作成
1.  **LP/トップページ:** サービスの特徴を動的なUIで訴求。
2.  **認証:** Googleログイン等でワンクリック登録/ログイン [3]。
3.  **新規構想ウィザード (Wizard Form):** [4]
    *   以下の項目を順に入力させる（選択式メイン）。
    *   ビジネスジャンル（例：エンタメ、人材、SaaS）
    *   ビジネスモデル（例：サブスクリプション、売り切り）
    *   売上目標（例：1億円未満）
    *   開始時期（例：6ヶ月以内）
    *   自分の強み（技術、営業、ブランド等）
    *   市場の課題、意思決定スタイル（直感/論理）、組織タイプ。
4.  **AIパートナー選択:** [4, 5]
    *   壁打ち相手の性格を選択（ロジカル型、チャレンジ型、メンター型、フレンド型など）。
    *   AIへの期待値設定（強みの抽出、リスク指摘など）。

### Phase 2: AIとの壁打ち (Core Feature)
*   **チャットインターフェース:**
    *   **音声入力機能 (必須):** 高精度な音声認識（Whisper API等）を用い、ユーザーが喋った内容をテキスト化して送信する。「喋って相談する」体験を重視 [5, 6]。
    *   **AIからの深掘り:** ユーザーのアイデアに対し、AIが質問や提案を行い、アイデアをブラッシュアップする [7]。

### Phase 3: 自動構造化 (Magic Feature)
*   **「AI入力」ボタン:** [8, 9]
    *   チャット画面の横（または別タブ）に「事業計画フォーム」を常設する。
    *   **機能:** チャットの会話履歴をAIが解析し、「サービス名」「サービス概要」「提供価値」「ターゲット市場」などの各項目をワンクリックで自動入力（Auto-fill）する。
    *   ユーザーは会話に集中し、まとまった段階でボタンを押すだけで計画書が埋まっていく体験を作る。

### Phase 4: アウトプット & 公開
*   **サムネイル設定:** キーワード（例："Love"）を入力するとAIが画像を検索・生成し、プロジェクトの表紙に設定できる [10]。
*   **事業計画書生成:** 入力されたデータを整形し、PDFとしてダウンロード可能にする [9]。
*   **評価レポート:** 事業の実現可能性やリスクをグラフ（レーダーチャート等）で可視化する [11]。
*   **公開プロジェクト (Timeline):** 作成した事業プランをサイト内で公開し、他ユーザー（またはAI）から評価をもらえるSNS的なタイムライン機能 [3, 11]。

## 4. 機能要件 (Functional Requirements)

### フロントエンド
*   **フレームワーク:** Next.js (React) + TypeScript 推奨。
*   **スタイリング:** Tailwind CSS + Framer Motion (「ビヨーン」というアニメーション実現のため)。
*   **音声認識:** Web Speech API または OpenAI Whisper API。

### バックエンド
*   **DB/Auth:** Supabase (PostgreSQL, Auth) または Firebase。
*   **AI連携:**
    *   OpenAI API (GPT-4o) または Claude 3.5 Sonnet API。
    *   System Promptにより、選択された人格（メンター型など）を演じ分けること。
*   **PDF生成:** `react-pdf` または `jspdf` 等を使用。

## 5. データモデル案 (Schema Concept)

*   **Users:** ID, Name, Email, Avatar
*   **Projects:**
    *   ID, UserID, Title, Status (Draft/Published)
    *   Attributes: Genre, Model, Goal, Strengths (JSON)
    *   PlanData: Overview, Target, Value, Competitors (Text)
    *   Analysis: Scores (Feasibility, MarketSize, etc.)
    *   Assets: ThumbnailURL
*   **ChatMessages:** ProjectID, Sender (User/AI), Content, Timestamp

## 6. 実装ステップ指示

1.  **環境構築:** Next.js + Tailwind + DBのセットアップ。
2.  **ウィザード実装:** 複数ステップのフォーム入力とデータ保存。
3.  **チャット基本実装:** テキストでのAI対話機能。
4.  **音声入力実装:** マイクボタンでの音声→テキスト変換。
5.  **AI入力機能 (重要):** 会話ログを要約して特定フィールドに流し込むAPIの実装。
6.  **UIリッチ化:** Framer Motion等でアニメーションを追加。
7.  **PDF出力:** 最終データのPDF化機能。
